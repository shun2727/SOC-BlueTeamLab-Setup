Task (based on NIST CSF 2.0 / SP 800-61 Rev. 3 )	: 

Task 1

	Title: Validate the alert
	Task group: Detect
	Description:

	Pull raw SSH auth logs for the alerting host: /var/log/auth.log (Debian/Ubuntu) or /var/log/secure (RHEL/CentOS), or SIEM sourcetype sshd/auth.

	Filter for failed logins in the alert window:
	sudo grep -E "Failed password|Invalid user" /var/log/auth.log | grep -E "<date>"

	Confirm brute-force pattern: many rapid failures from one/few source IPs, multiple usernames from same IP, high frequency (e.g. >10 attempts in 1-2 min).

	Rule out false positives: monitoring/backup scripts, misconfigured clients, isolated user typos.

	Log finding in the case: "Alert validated as brute force" or "False positive — close." If false positive, close case here.

Task 2

	Title: Identify source IP(s)
	Task group: Detect
	Description:

	Extract distinct attacker IPs:
	sudo grep "Failed password" /var/log/auth.log | awk '{for(i=1;i<=NF;i++) if($i=="from") print $(i+1)}' | sort -u

	SIEM equivalent: sourcetype=sshd failed_login=true | stats count by src_ip

	Check reputation (AbuseIPDB/GreyNoise/VirusTotal). Document IP list, attempt counts, time range.

Task 3

	Title: Determine scope
	Task group: Detect
	Description:

	Check whether other hosts show failed logins from the same source IP(s):
	SIEM query: sourcetype=sshd failed_login=true src_ip=<ip> | stats count by host

	List all affected hostnames/IPs. If multiple hosts are hit (credential spray), raise case severity from Medium to High.

Task 4

	Title: Check for successful authentication (critical pivot point)
	Task group: Respond
	Description:

	Search for successful logins from the attacker IP(s):
	sudo grep -E "Accepted password|Accepted publickey" /var/log/auth.log | grep -E "<attacker_ip>"

	Also check session opens:
	sudo grep "session opened for user" /var/log/auth.log | grep -E "<attacker_ip>"

	If ANY success is found: escalate severity to High immediately, notify L2, and proceed to Tasks 6-7 as a full incident. If no success: proceed to routine closure after hardening (Tasks 10-12).

Task 5

	Title: Identify targeted account(s)
	Task group: Respond
	Description:

	Extract attempted usernames:
	sudo grep "Failed password" /var/log/auth.log | grep -E "<attacker_ip>" | awk '{for(i=1;i<=NF;i++) if($i=="user") print $(i+1)}' | sort | uniq -c | sort -rn

	Categorize as default/common (root, admin, ubuntu, user) vs. real employee usernames — the latter suggests a targeted attack, not opportunistic scanning. Document list with frequency.

Task 6

	Title: Check for lateral movement (mandatory if Task 4 confirms success)
	Task group: Respond
	Description:

	Identify session start time, then review:
	- Command history: ~/.bash_history, /root/.bash_history
	- If auditd enabled: ausearch -i --start <time> --end <time>
	- New SSH keys: ls -la ~/.ssh/authorized_keys
	- New accounts: cat /etc/passwd or getent passwd | cut -d: -f1
	- Privilege escalation: grep sudo /var/log/auth.log
	- Outbound connections: sudo netstat -tupn or sudo ss -tupn

	Document all findings in the case.

Task 7

	Title: Check for persistence mechanisms
	Task group: Respond
	Description:

	Check the following for attacker-planted persistence:
	- Cron: cat /etc/crontab, ls -la /etc/cron.*, and per-user: for user in $(cut -d: -f1 /etc/passwd); do crontab -u $user -l 2>/dev/null; done
	- systemd timers: systemctl list-timers --all
	- SSH keys: review ~/.ssh/authorized_keys for unknown entries
	- sshd_config: sudo cat /etc/ssh/sshd_config — check AllowUsers, PermitRootLogin
	- Running services: systemctl list-units --type=service --state=running

	Remove any suspicious cron jobs, timers, keys, or services found. Log what was removed.

Task 8

	Title: Contain — block source IP(s)
	Task group: Respond
	Description:

	Block at firewall/security group/WAF, e.g.:
	sudo iptables -A INPUT -s <attacker_ip> -j DROP

	If Fail2Ban available, confirm running:
	sudo systemctl enable fail2ban && sudo systemctl start fail2ban
	sudo fail2ban-client status sshd

	Log the rule/action taken, IP, and timestamp.

Task 9

	Title: Contain — reset compromised account(s) (mandatory if compromise confirmed)
	Task group: Respond
	Description:

	Force password reset: sudo passwd -e <username>
	Revoke/rotate SSH keys in ~/.ssh/authorized_keys
	Kill active sessions: sudo pkill -u <username>

	If host compromise is confirmed: isolate the host from the network and initiate forensic review before further use.

Task 10

	Title: Harden and restore service
	Task group: Recover
	Description:

	Edit /etc/ssh/sshd_config:
	- PermitRootLogin no
	- PasswordAuthentication no / PubkeyAuthentication yes
	- AllowUsers <known users>

	Configure Fail2Ban jail (/etc/fail2ban/jail.conf):
	[sshd]
	enabled = true
	port = ssh
	filter = sshd
	logpath = /var/log/auth.log
	maxretry = 5
	bantime = 600

	Restrict SSH exposure to VPN/bastion only via security group rules. Restart: sudo systemctl restart sshd

Task 11

	Title: Validate remediation
	Task group: Recover
	Description:

	Confirm IP block is active: iptables -L or SIEM block status
	Confirm no persistence remains: re-check cron, systemd, SSH keys, sshd_config
	Confirm service healthy: sudo systemctl status sshd
	Monitor for further attempts via SIEM dashboard / Fail2Ban logs.
	Document validation results in the case.

Task 12

	Title: Write closing summary and recommend follow-ups (mandatory)
	Task group: Improvement
	Description:

	Complete the Incident Report page: timeline, source IPs/usernames/techniques, impact (compromise Y/N), actions taken.

	Recommend detection improvements: lower failed-SSH alert threshold, add correlation rule (failed SSH followed by success), tune SIEM based on observed pattern.

	Recommend broader fixes: enforce MFA on critical systems, centralize logging to SIEM, standardize SSH hardening fleet-wide.

	Store report in IR platform/knowledge base for future reference.
